Inclusão de Ficheiros Externos

Esta secção demonstra a inclusão de diferentes tipos de ficheiros externos.

1. Folha de Estilos CSS Externa

A tag <link rel="stylesheet" href="style.css"> no <head> carrega um ficheiro CSS externo.

A tag <link> bloqueia a renderização da página?
Sim. O browser bloqueia a renderização até que todos os ficheiros CSS via <link rel="stylesheet"> sejam descarregados e processados, evitando o FOUC (Flash of Unstyled Content). Para CSS não crítico podem usar-se técnicas como rel="preload" ou media="print".

2. Script JavaScript Externo

A tag <script src="script.js" async> no <head> carrega um ficheiro JavaScript externo.

async vs defer:
• async: descarregado em paralelo e executado imediatamente quando disponível, podendo interromper o parsing. Ideal para scripts independentes (ex: analytics, publicidade).

• defer: descarregado em paralelo mas executado apenas após o HTML estar completamente analisado, mantendo a ordem entre scripts. Ideal para scripts que dependem do DOM ou de outros scripts (ex: frameworks, bibliotecas de UI).

3. Imagem Externa com Lazy Loading

O atributo loading="lazy" adia o carregamento da imagem até que esta esteja próxima da área visível (viewport).

Quando usar lazy loading?
• Em páginas com muitas imagens (galerias, feeds, lojas online).
• Em imagens abaixo da dobra (below the fold), fora da área visível inicial.
• Para reduzir o tempo de carregamento inicial e o consumo de dados.
• Não usar em imagens críticas acima da dobra (hero images, logótipos), pois causaria atrasos visíveis.

4. Vídeo Externo

A tag <video> permite incorporar vídeos diretamente na página HTML.

O atributo preload="metadata" instrui o browser a carregar apenas os metadados do vídeo (duração, dimensões) sem descarregar o ficheiro completo.
