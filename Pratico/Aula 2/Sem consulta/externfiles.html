A tag <link> bloqueia a renderização da página? Lazy Loading de imagens: quando deve ser utilizado? Script assíncrono (async) vs diferido (defer): quando usar cada um?
Sim. A tag <link rel="stylesheet"> bloqueia a renderização da página (render-blocking) enquanto o ficheiro CSS é descarregado e processado. Este comportamento é intencional para evitar o FOUC (Flash of Unstyled Content), ou seja, a página aparecer momentaneamente sem estilos. Por isso, os ficheiros CSS devem sempre ser incluídos no <head>. O loading="lazy" deve ser usado em imagens que estão fora da área visível inicial da página (abaixo do "fold"). O browser só as carrega quando o utilizador se aproxima delas ao fazer scroll, poupando largura de banda e acelerando o carregamento inicial. Não deve ser usado em imagens críticas visíveis logo no topo da página (ex: logótipos, banners acima do fold), pois isso atrasaria o carregamento de conteúdo importante. async: o script é descarregado em paralelo com o HTML e executado assim que estiver disponível, interrompendo o parsing. Ideal para scripts independentes que não dependem do DOM nem de outros scripts (ex: Google Analytics, publicidade).

defer: o script é descarregado em paralelo, mas só é executado após o HTML estar completamente processado, respeitando a ordem de declaração. Ideal para scripts que manipulam o DOM ou que dependem de outros scripts (ex: lógica da aplicação, bibliotecas como jQuery).
